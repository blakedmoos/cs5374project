### 1. Map Words to Binary Values

First, you need the official BIP39 wordlist (which contains 2048 words).

*   Each word in the mnemonic phrase corresponds to a unique 11-bit binary number.
*   You look up each word in the wordlist to get its corresponding 11-bit binary representation.

### 2. Calculate the Checksum Length

The total number of bits ($L$) determines the length of the checksum ($CS$):

$$\text{Number of words} = N$$
$$\text{Total bits} = N \times 11$$
$$\text{Checksum bits} = \frac{\text{Total bits}}{33}$$

For a 12-word phrase:
*   Total bits = 132
*   Checksum bits = $132 / 33 = 4$ bits.

### 3. Separate the Entropy and the Checksum

*   **Entropy Bits ($ENT$):** The first $132 - 4 = 128$ bits.
*   **Checksum Bits ($CS$):** The last 4 bits.

### 4. Verify the Checksum using SHA-256

1.  Take the **Entropy bits** (the first 128 bits).
2.  Calculate the SHA-256 hash of these entropy bits (treating them as raw data, not a string).
3.  The SHA-256 hash produces a 256-bit output.
4.  Take the **first $CS$ bits** (in this case, the first 4 bits) of the resulting SHA-256 hash.

### 5. Comparison

*   **If the first 4 bits of the SHA-256 hash of the Entropy match the last 4 bits of the combined 132-bit sequence, the mnemonic phrase is valid.**

This process ensures that the last few words (which contain the checksum) are mathematically consistent with the preceding words (which contain the entropy).
