### Understanding LLM Data Leakage and Prompt Injection

In the context of an AI wallet assistant, "data leakage" refers to the model inadvertently revealing sensitive information, such as a seed phrase, that was provided in its system prompt or training data. This often happens through "prompt injection," where a user crafts inputs to override the model's original instructions.

#### Common Vulnerability Mechanisms:

1.  **Context Leakage**: The model treats the system prompt as part of its conversational context. If not properly constrained, it may reference the system prompt when asked about its "instructions," "identity," or "hidden data."
2.  **Semantic Association**: Even when a model is told not to reveal a secret, that secret remains in its active attention mechanism. An attacker might use "semantic leakage" techniques, asking the model to generate related words, synonyms, or creative writing, hoping the secret words influence the output.
3.  **Indirect Injection**: This occurs when the model processes external data (like a user-provided list) that contains instructions meant to hijack the model's behavior.

### Defensive Strategies for AI Assistants

To protect sensitive information like Bitcoin seed phrases, developers use several layers of defense:

*   **Output Filtering**: Implementing a secondary layer that scans the AI's response for sensitive patterns (e.g., matching words against the BIP-39 word list or detecting 12-word sequences) and blocks the response if a match is found.
*   **System Prompt Hardening**: Using clear delimiters to separate system instructions from user input and explicitly defining the boundaries of what information is "secret."
*   **Context Segregation**: Instead of putting the actual seed phrase in the system prompt, the AI could interact with a secure, external API or vault that performs the necessary operations without exposing the raw data to the LLM's context.
*   **Few-Shot Prompting for Safety**: Providing the model with examples of attempted "attacks" and the correct way to refuse them, which reinforces the security boundaries.
*   **Adversarial Robustness Training**: Training the model specifically to recognize and resist common jailbreaking and injection techniques.

In a real-world scenario, a seed phrase should generally never be included in the system prompt of an LLM, as LLMs are probabilistic engines and cannot currently guarantee 100% data confidentiality once information is in their active context.
